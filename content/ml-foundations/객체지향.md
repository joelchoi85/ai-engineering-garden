관련된 데이터(속성)와 그걸 다루는 기능(메소드)을 하나로 묶는 프로그래밍 방식(OOP).

**왜 필요한가?** 학생 정보를 변수로만 다루면 `name_1, age_1, scores_1, name_2, ...`처럼 학생이 늘 때마다 변수가 폭발적으로 늘어난다.
대신 `Student`라는 **설계도(클래스)** 를 한 번 만들고, 그걸로 객체(인스턴스)를 찍어내면 깔끔하다.

```python
class Student:
    def __init__(self, name, scores):  # 생성자
        self.name = name               # 속성
        self.scores = scores
    def average(self):                 # 메소드
        return sum(self.scores)/len(self.scores)
```

**파이썬은 처음부터 객체였다.** `my_list.append(5)`에서 그 점(`.`)이 "이 객체야, 네 메소드 실행해"라는 뜻 — 리스트·문자열·딕셔너리가 다 객체다. 클래스를 배운다는 건 내가 직접 그런 타입을 만드는 것.
