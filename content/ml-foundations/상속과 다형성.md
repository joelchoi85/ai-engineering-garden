[[객체지향]]의 두 핵심 도구.

**상속(Inheritance)**: 공통 기능을 부모 클래스에 두고 자식이 물려받아 중복을 없앤다. `Animal`을 만들고 `Dog`, `Cat`이 상속받으면 공통 코드를 다시 안 짜도 된다.

**다형성(Polymorphism)**: 같은 메소드 이름을 자식마다 다르게 구현하고, 부모 타입으로 묶어 호출하면 각자 알맞게 동작한다.

```python
for animal in [Dog(), Cat()]:
    animal.speak()   # 개는 "멍멍", 고양이는 "야옹"
```

이 패턴이 딥러닝 코드의 기반이다 — PyTorch 모든 모델이 `nn.Module`을 **상속**하고([[nn.Module]]), sklearn의 `fit`/`predict`가 모델 종류와 무관하게 통일된 건 다형성 덕분이다([[결정 트리]]도 [[선형 회귀]]도 같은 인터페이스).
